使用方法

STEP0: 署名鍵を生成する (操作: 送信者)

  1. 送信者は、それぞれ 署名用鍵生成ボタン を押し、処理を開始します。
  2. 処理が完了すると、2個のファイルがダウンロードされます。(許可を要求された場合は許可します)
    • {乱数}.sign.pub : 公開鍵 (公開しても問題ない)
    • {乱数}.sign.sec : 秘密鍵 (絶対に公開してはいけない)

STEP1: 暗号化鍵を生成する (操作: 送信者/受信者)

  1. 送信者と受信者は、それぞれ 鍵生成 ボタンを押し、処理を開始します。
  2. 秘密鍵を保護するためのパスワード設定を求められるので、推測されにくい値を入力します。
  3. 処理が完了すると、2個のファイルがダウンロードされます。(許可を要求された場合は許可します)
    • {乱数}.pub : 公開鍵 (公開しても問題ない)
    • {乱数}.sec : 秘密鍵 (絶対に公開してはいけない)

STEP2: 公開鍵を交換する (操作: 送信者/受信者)

  1. 送信者と受信者は、それぞれ生成した {乱数}.pub を、電子メールなどで相互に交換します。

STEP3: 暗号化を行う (操作: 送信者)

  1. 送信者は、ツールの各フォームへファイルを入力します。
    • データ : 暗号化したいファイル (複数指定可能)
    • 公開鍵 : {乱数}.pub (受信者から受け取ったもの)
    • 秘密鍵 : {乱数}.sec (自身で生成したもの)
  2. 暗号化 ボタンを押し、処理を開始します。
  3. 秘密鍵のパスワードを求められるので、鍵生成のときに設定したパスワードを入力します。
  4. 処理が完了すると、1個のファイルがダウンロードされます。
    • cipher.bin : 暗号データ (公開しても問題ない)

STEP4: 暗号データを送信する (操作: 送信者)

  1. 送信者は、暗号化された cipher.bin を、電子メールなどで受信者へ渡します。

STEP5: 暗号データを復号する (操作: 受信者)

  1. 受信者は、ツールの各フォームへファイルを入力します。
    • データ : cipher.bin (送信者から受け取ったもの)
    • 公開鍵 : {乱数}.pub (送信者から受け取ったもの)
    • 秘密鍵 : {乱数}.sec (自身で生成したもの)
  2. 復号 ボタンを押し、処理を開始します。
  3. 秘密鍵のパスワードを求められるので、鍵生成のときに設定したパスワードを入力します。
  4. 処理が完了すると、元のファイルがダウンロードされます。
技術仕様
  • 実装: Web Crypto API
  • 鍵交換: ECDH
  • 公開鍵: NIST P-521 - 521bit Key
  • 共通鍵: AES-GCM - 256bit Key, 128bit Tag, 128bit IV
安全性
  • データ処理はローカルで完結するため、インターネット通信を伴う処理はありません。
  • サードパーティ製のライブラリは使用していません。
  • MITライセンス準拠のオープンソースソフトウェアとなります。
  • ハッシュ値: SHA512
