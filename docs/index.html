ドキュメント

使用方法

暗号化が可能なデータは、ZIPアーカイブ .zip のみとなります。
送信したいデータは、予め通常のZIP圧縮(非暗号化)を行う必要があります。

  1. 送信者と受信者は、それぞれ "鍵生成" ボタンを押し鍵ペアを生成する。
  2. 鍵生成が完了すると、2個のファイルがダウンロードされる。(ダウンロードの際に許可を求められる場合があります)
    • public.key
    • secret.key
  3. 送信者と受信者は、それぞれ生成した public.key を電子メール等で相互に交換する。
  4. 送信者は、当ツールの各フォームへファイルを入力する。
    • 暗号化したい .zip: "データ" フォーム
    • 受信者から入手した public.key: "公開鍵" フォーム
    • 自身で生成した secret.key: "秘密鍵" フォーム
  5. "暗号化/復号" ボタンを押し、処理を開始する。
  6. 処理が完了すると、1個のファイルがダウンロードされるので、電子メール等で受信者へ伝送する。
    • cipher.bin
  7. 受信者は、当ツールの各フォームへファイルを入力する。
    • 送信者から入手した cipher.bin: "データ" フォーム
    • 送信者から入手した public.key: "公開鍵" フォーム
    • 自身で生成した secret.key: "秘密鍵" フォーム
  8. "暗号化/復号" ボタンを押し、処理を開始する。
  9. 処理が完了すると、1個のファイルがダウンロードされるので、通常のZIP展開(非暗号化)を行う。
    • raw.zip
技術仕様
  • 実装: Web Crypto API
  • 鍵交換: 楕円曲線ディフィーヘルマン鍵共有
  • 公開鍵暗号: 楕円曲線暗号 - NIST P-521
  • 共通鍵暗号: AES-GCM - 256bit Key, 128bit Tag, 128bit IV
安全性

アプリケーションの改竄検出を可能にするため、ハッシュ値を公開しています。
全ての処理がオフラインで完結するため、データを外部へ送信することはありません。
サードパーティのJavaScriptライブラリは使用していません。
MITライセンス準拠のオープンソースソフトウェアとなります。
